# Handoff Template

Use this at the end of substantial work so the next human or LLM contributor
can continue cleanly.

This is an authoritative repo-local handoff file.
Use it to record what actually changed, what was validated, and what remains.

Read this together with `AGENTS.md` and
`docs/project/implementation-tracker.md`.
It improves continuity for humans and LLMs, but it does not guarantee the next
contributor will follow every instruction perfectly.

## Current Result

- what changed
- what was validated
- what remains intentionally unimplemented

## Docs Updated

- list the docs updated in the same change

## Verification

- list checks or tests performed
- state what was not verified

## Next Valid Steps

1. recommend the narrowest next step

## Remaining Non-Goals

- restate what should still not be built yet
