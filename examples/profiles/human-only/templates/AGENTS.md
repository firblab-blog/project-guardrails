## Read This First

Before substantial work, read:

1. `AGENTS.md`

## Human Review Contract

- keep the current approved scope explicit before widening work
- write changes so another human reviewer can understand the intent quickly
- update repo-local docs when the approved plan changes
- leave a handoff summary that another contributor can continue without guesswork

## Non-Goals

- do not assume an automated agent will enforce every workflow detail
- do not widen repo policy without explicit human approval
