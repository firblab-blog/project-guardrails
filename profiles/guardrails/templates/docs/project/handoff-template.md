# Handoff Template

Use this after substantial work so the next contributor can restart with the
repo's actual state, not a guess.

## Completed In This Slice

- summarize the concrete code or doc changes
- call out any profile/template content that changed

## Validation

- list the commands, tests, or manual checks that were run
- note any validation that was intentionally skipped

## Docs And State Updated

- list the repo-local docs updated in the same change
- note whether the implementation tracker or decision log changed

## Next Safe Step

1. name the narrowest follow-up step that is still aligned with the tracker

## Risks Or Watch Items

- list anything the next contributor should verify before widening scope
